    列选择

    概述

    列选择可以用来选择一个文件中的矩形区域。 列选择不通过一个单独的模式运作，而是使用多重选择。 你可以添加选取来选择文本的多个代码块或者减少选取去掉一个代码块。

    使用鼠标

    不同平台使用不同的鼠标键和键盘键:

    OSX

    • 鼠标左键+Option
    • 或者鼠标中键
    • 添加至选区：Command
    • 从选区减少: Command + Shift

    Windows

    • 鼠标右键 + Shift
    • 或者鼠标中键
    • 添加至选区: Ctrl
    • 从选区减少: Alt

    Linux

    • 鼠标右键 + Shift
    • 添加至选区: Ctrl
    • 从选区减少: Alt

    使用键盘

    OSX

    • Ctrl + Shift + Up
    • Ctrl + Shift + Down

    Windows

    • Ctrl + Alt + Up
    • Ctrl + Alt + Down

    Linux

    • Ctrl + Alt + Up
    • Ctrl + Alt + Down

    用键盘多重选择

    添加一行

    要向选区添加之上和之下的一行,使用Ctl+Alt+Up 和 Ctrl+Alt+down (OS X: Ctrl+Shift+Up 和 Ctrl+Shift+Down)。

    如果你走的太远了,使用撤销选区(Ctrl+U， 或者 Command+U 在OS X上)来进行回退。 将选区转换成多个单行选区

    选择一个多行的代码块，然后将其转换为每行一个选区的多个选区，这时候要用Ctrl+Shift+L, 或 Command+Shift+L 在OS X上.

    快速添加下一个

    为了添加下一个出现的单词到选区，使用快速添加下一个,它绑定到Ctrl+D 在Windows和Linux, 或者 Command+D 在OS X上.

    重申,如果你走的太远了,使用撤销选区(Ctrl+U， 或者 Command+U 在OS X上)来进行回退。

    全选

    要添加所有出现的符合选区要求的词到选区，使用全选: Alt+F3在Windows和Linux,或者 Ctrl+Command+G在OS X上.

    单选区

    要从多重选择回到一个单选区,按Escape.

    自动完成

    概述

    当你键入完一个键时自动完成会显示可完成的列表，因此你可以只敲几个键完成一些个长的单词。 它在源码和html代码中是默认启用的(但只会在键入 < 字符后起作用).

    禁用自动完成

    通过auto_complete配置可以禁用自动完成。要禁用的话,把下面这行加入到 Preferences/File Settings - User: 

    "auto_complete": false

    在自动完成被禁用的情况下,可以手动显示完成层,或者tab直接插入完成中最靠前的那项而不显示完成列表。

    手动显示完成

    按Ctrl+Space将会显示完成弹层如果他没被显示的话。如果正在显示,他将选择下一项。

    使用tab来提交

    默认的，完成中的选中项目将在你键入回车时生效。这会造成完成标签和插入新的一行之间的歧义。同时设置 auto_complete_commit_on_tab 配置为true,回车只会插入新的一行，并且tab将会补全当前的自动完成。这还有其他的好处：因为Sublime Text知道这儿没有歧义，他可能显示更多相关完成的列表，并且极有可能把你想要的排在第一位。

    推荐启用用tab提交，但是这将话费你一点时间去习惯。

    Tab完成

    概述

    Tab完成允许通过按tab键快速的完成单词。当启用时按下tab编辑器会使用它的模糊匹配算法展开光标左侧的文本成最佳匹配项。

    Tab完成默认是启用的。

    禁用Tab完成

    有时候tab完成不是很需要。要关闭他的话，加入下面这行到Preferences/File Settings - User: 

    "tab_completion": false

    插入原生tab

    要插入tab而不是补全，按shift+tab.

    改变完成

    有时候完成的文本不是你想要的：要更换他，按ctrl+space。这将还原完成，并显示标准的自动完成列表。

    另外，tab可以再次按下去循环下一个完成。

    完成源码

    关于完成的源码，和创建自己的完成可以访问 sublimetext.info .

    自由模式（沉浸模式）

    概述

    自由模式（沉浸模式）全屏显示你的文件，你的显示器当中除了文本啥也没有。所有的UI chrome是隐藏的，但是可以访问。沉浸模式可以通过View/Enter Distraction Free Mode菜单项进入。

    当处于沉浸模式时，所有的UI chrome (侧边栏,迷你地图,状态栏,等)将被隐藏。你可以通过View菜单选择性的选择部分UI-你的配置将会在下次进入沉浸模式时生效。

    定制化

    部分配置将会在进入沉浸模式时应用。这些默认的配置位于Packages/Default/Distraction Free.sublime-settings)他们是: 

    {
  "line_numbers": false,
  "gutter": false,
  "draw_centered": true,
  "wrap_width": 80,
  "word_wrap": true,
  "scroll_past_end": true
}

    你可以定制这些通过编辑Packages/User/Distraction Free.sublime-settings，这可以通过Preferences/File Settings - More 菜单访问到。

    上面值得注意的是wrap_width设置。它的值80造成超过80字符时换行。你也许想把它设的高点，或者在窗口宽度是换行（设为0的时候）。

    复古模式(vi模式)

    概述

    复古是一个Sublime Text 2 的vi模式编辑包。它允许你结合vi的命令行模式和Sublime Text的特性，包括多重选择。

    复古模式在开放开发和补丁方面更受欢迎。如果你想要贡献，详情可见GitHub页。 启用复古模式

    复古模式默认是禁用的，通过ignored_packages配置。 如果你把"Vintage"忽略包列表中移除，你就可以使用vi快捷键编辑了:

    1. 选中Preferences/Settings - Default菜单项

    2. 编辑ignored_packages配置，把他从:

    3. "ignored_packages": ["Vintage"]

    改成:

    "ignored_packages": []

    然后保存文件。

    1. 现在复古模式就启用了 - 你会看见"INSERT MODE"在任务栏上列出来了。

    复古模式默认就是插入模式。这可以通过添加以下来改变:

    "vintage_start_in_command_mode": true

    来使用自己的配置。

    它包括了

    复古模式包括了最基本的操作： d (delete), y (copy), c (change), gu (lower case), gU (upper case), g~ (swap case), g? (rot13), < (unindent), and > (indent).

    它也包含了许多动作，包括l, h, j, k, W, w, e, E, b, B, alt+w (move by sub-words), alt+W (move backwards by sub-words), $, ^, %, 0, G, gg, f, F, t, T, ^f, ^b, H, M, and L.

    文本对象是支持的，包括文字，引号，括号和标签。

    那里重复的('.')，作为被指定命令和动作的计数。 寄存器就像宏和书签是支持的。许多杂项命令也是被支持的，比如 *, /, n, N, s, S 和其他的。

    它没有

    插入模式是通常Sublime Text 2编辑,通过通常Sublime Text 2 绑定的键位: vi 插入模式键位没有实现。

    Ex 命令行 没有实现，除了:w and :e, 其他通过控制面板。 实现的原理

    复古模式完全是靠快捷键和插件API实现的 - 随意浏览复古包，看看他们是如何组织的。举个例子，你想绑定"jj"到退出插入模式，你可以添加这个快捷键: 

    { "keys": ["j", "j"], "command": "exit_insert_mode",
  "context":
  [
    { "key": "setting.command_mode", "operand": false },
    { "key": "setting.is_widget", "operand": false }
  ]
}

    OS X Lion

    在狮子中，按住一个键不会重复作用这个键，而是显示出一个弹出菜单供选择字符间的变化。这不能很好的和命令模式工作，因此你也许想禁用它。只要通过在终端里运行下面这句: 

    defaults write com.sublimetext.2 ApplePressAndHoldEnabled -bool false

    Ctrl键组

    复古模式支持这些Ctrl的绑定:

    • Ctrl+[: Escape
    • Ctrl+R: Redo
    • Ctrl+Y: Scroll down one line
    • Ctrl+E: Scroll up one line
    • Ctrl+F: Page Down
    • Ctrl+B: Page Up

    然而，因为他们和其他Sublime Text 2 的快捷键冲突，他们在Windows and Linux上默认是禁用的。他们可以通过设置the vintage_ctrl_keys配置来启用: 

    "vintage_ctrl_keys": true

    项目

    概述

    Sublime Text 2中的项目是由2个文件构成: sublime-project文件， 它包含了项目定义；还有sublime-workspace文件,它包含了用户指定的数据比如打开的文件以及它们的每个修改之处。

    按照惯例，sublime-project文件会被迁入到版本控制中，而sublime-workspace文件则不会。 项目格式

    sublime-project文件是JSON格式，并且有三个顶级对象: folders,用来包含目录, settings, 文件配置的复写，和build_systems，给项目指定编译系统。下面是个例子: 

    {
    "folders":
    [
        {
            "path": "src",
            "folder_exclude_patterns": ["backup"]
        },
        {
            "path": "docs",
            "name": "Documentation",
            "file_exclude_patterns": ["*.css"]
        }
    ],
    "settings":
    {
        "tab_size": 8
    },
    "build_systems":
    [
        {
            "name": "List",
            "cmd": ["ls"]
        }
    ]
}

    目录

    每个目录必须有一个path， 并且可选的有一个folder_exclude_patterns和file_exclude_patterns配置。path也许相对于项目目录，或者一个全路径。 目录也许也有一个name配置，用来设置在侧边栏显示的名字。

    早期版本转换的项目也许会有一个mount_points入口在项目目录下。如果你想使用排除匹配模式，你需要将原入口改成上述格式。

    配置

    这里的Settings可以使用settings键值指定，并且覆盖普通用户的配置。 注意，他们不能覆盖语法的配置。

    编译系统

    Build Systems指定了一组 编译系统 定义。为了区别普通编译系统，每个编译系统必须指定一个name。 这里列出的编译系统将在普通的Tools/Build Systems菜单上可见。

    配置

    概述

    Sublime Text 2有许多配置来定制它的行为。通过修改文件来改变配置：尽管这比用一个GUI图形界面有点小麻烦，但是你被奖励了一个灵活的系统。 配置

    想了解哪些配置是可用的，每个配置的说明，请看下这个Packages/Default/Preferences.sublime-settings。你可以通过Preferences/Settings - Default菜单打开这个文件。

    当你发现一些配置要更改的时候，把他们加入到你的用户配置里去 (通过Preferences/Settings - User 菜单可以访问到)，这样在升级时，他们可以被保留下来。

    配置文件

    配置文件将以一下顺序载入：

    1. Packages/Default/Preferences.sublime-settings
    2. Packages/Default/Preferences ( ).sublime-settings
    3. Packages/User/Preferences.sublime-settings
    4. <Project Settings>
    5. Packages/ / .sublime-settings
    6. Packages/User/ .sublime-settings
    7. <Buffer Specific Settings>

    总之，你应该把你的配置放到Packages/User/Preferences.sublime-settings中。如果你想指定某种文件类型的配置，比如，Python，你应该把他们放在Packages/User/Python.sublime-settings中。

    示列配置文件

    试着保存为Packages/User/Preferences.sublime-settings

    {
    "tab_size": 4,
    "translate_tabs_to_spaces": false
}

    每个语法配置

    配置也许会为每个语法去指定。通常这样的用户想要给每个文件类型不同的缩进或者语法高亮颜色。

    你可以给当前语法编辑这些配置通过Preferences/Settings - More/Syntax Specific - User菜单。

    每个项目配置

    可以设置一个每个项目基础配置，详细参考Project Documentation

    沉浸模式配置

    Distraction Free Mode有一个额外的配置文件名称为(Distraction Free.sublime-settings)。你可以设置只有沉浸模式有用的文件配置-通过Preferences/Settings - More/Distraction Free - User菜单访问到。

    通过快捷键改变设置

    toggle_setting命令可以被用来切换设置。比如，给切换word_wrap当前文件设置一个快捷键，你可以使用(在Preferences/Key Bindings - User中的): 

    {
    "keys": ["alt+w"],
    "command": "toggle_setting",
    "args":
    {
        "setting": "word_wrap"
    }
}

    set_setting命令可以被用来给某个配置指定一个特殊的值。比如，这个快捷键是当期文件使用Cobalt color scheme语法高亮: 

    {
    "keys": ["ctrl+k", "ctrl+c"],
    "command": "set_setting",
    "args":
    {
        "setting": "color_scheme",
        "value": "Packages/Color Scheme - Default/Cobalt.tmTheme"
    }
}

    这里修改的配置是缓冲区的配置：他们重载配置文件中的配置，但是只对当前文件起作用。

    故障排除

    由于有好多处可以设置配置的地方，有时候看到当前文件实际使用的配置很有用。你可以在终端里操作以下来查看设置:

    view.settings().get('font_face')

    改变字体

    概述

    可以通过添加这几行到Preferences/Settings - User:

    "font_face": "Courier New",
"font_size": 10

    改变字体，当你保存时字体就会变了。

    你可以通过浏览 Preferences/Settings - Default来查看其它要改变的配置。

    缩进

    概述

    缩进配置决定了tab停留的大小并且控制是否tab键应该插入tab还是空格。此外，因为自动检测的缘由，他们可以为全局、每个文件类型或者每个文件设置。 配置 tab_size 整数。一个tab和空格相等的数量

    translate_tabs_to_spaces 布尔，如果真，当tab键按下时，空格将会被插入到这个新tab停止的地方，而不是插入一个tab字符

    detect_indentation 布尔，如果真（默认是），当载入一个文件时会自动计算它的tab_size和translate_tabs_to_spaces

    use_tab_stops 布尔，如果translate_tabs_to_spaces为真，use_tab_stops将使tab和backspace的insert/delete提到下一个制表位

    配置文件

    配置文件将以一下顺序载入:

    1. Packages/Default/Preferences.sublime-settings
    2. Packages/Default/Preferences ( ).sublime-settings
    3. Packages/User/Preferences.sublime-settings
    4. Packages/ / .sublime-settings
    5. Packages/User/ .sublime-settings

    一般来说，你应该把你的配置放入Packages/User/Preferences.sublime-settings。如果你想给某个文件类型指定配置,举个例子，Python，你应该把他们放入Packages/User/Python.sublime-settings.

    示列配置文件

    试着保存为Packages/User/Preferences.sublime-settings

    {
  "tab_size": 4,
  "translate_tabs_to_spaces": false
}

    每个语法配置

    可以设定一个语法配置。你可以给当前语法设置配置通过编辑Preferences/Settings - More/Syntax Specific - User菜单下出现的文件。

    缩进检测

    当一个文件被载入时，它的文档被审查，并且tab_size和translate_tabs_to_spaces配置将设给那个文件。当这些发生时状态栏区域将会报告这个结果。尽管这事 通常 进行的很好，你也许会想禁用它。你可以改detect_indentation配置实现它。

    Indentation detection可以手动执行通过View/Indentation/Guess Settings From Buffer菜单，这个会执行detect_indentation命令。

    Tabs和空格之间的转换

    View/Indentation有把当前文件中的tabs和空格之间互相转换的命令。这些菜单项将会运行expand_tabs和unexpand_tabs命令。

    自动缩进

    当你按下回车时自动缩进猜测前导空格的数量来插入到每一新生行。它通过以下配置控制: auto_indent 布尔，默认启用。启用自动缩进。 smart_indent 布尔，默认启用。使自动缩进更智能一点，e.g.在C中声明了if后下一行会自动缩进一次。 trim_automatic_white_space 布尔，默认启用。当移动插入符移出一行时去除auto_indent生成的白空格。 indent_to_bracket 布尔，默认禁用。缩进时添加白空格到第一个闭包（bracket翻译不准谷歌翻译成托架了，自己理解吧）单词。使用缩进时会像下面这样： 

    use_indent_to_bracket(to_indent,like_this);

    拼写检查

    概述

    Sublime Text使用Hunspell 作为它的拼写检查器。可替换的字典可以从OpenOffice.org Extension List获取到。

    字典

    Sublime Text目前只支持UTF-8编码的字典。大多数给问题语言的字典不是用UTF-8编码的而是使用一种更为紧凑的编码。要在Sublime Text中使用一个字典，首先需要把它转成UTF-8。

    一旦你有了一个用UTF-8编码的字典，他就可以被放到一个包里来安装，举个例子， Packages/User,这个你可以从 Preferences/Browse Packages 菜单项中浏览到。一旦文件放好了模拟就可以从 View/Dictionary 菜单的设置中选择字典了。

    配置

    我们有2个 配置 影响到拼写检查: spell_check,这控制着拼写检查的启用与否和 dictionary, 这提供了要使用的字典文件的路径。举个例子: 

    "spell_check": true,
"dictionary": "Packages/Language - English/en_US.dic"

    命令 

    next_misspelling: 选择下一个错误拼写
prev_misspelling: 选择上一个错误拼写
ignore_word: 添加所选单词通过word参数到ignore list（忽略列表）。

    包

    概述

    包是被Sublime Text使用的资源文件集合：如插件、语法高亮定义、菜单、Snippet等。Sublime Text附带几个包，其他更多的是用户自己创建的。

    包被存储在.sublime-package后缀的文件中，其实是zip文件改了个后缀名。包也可以被存储在一个被解压后的目录里，或者二者都有：任何在包目录里的松散文件将会覆盖存储在以.sublime-package后缀的文件。

    位置 Zipped过的包可以存放在:

    • <executable_path>/Packages
    • <data_path>/Installed Packages

    松散的包会被存放在:

    • <data_path>/Packages

    举个例子, 包Python存储在 <executable_path>/Packages/Python.sublime-package,并且任何放在<data_path>/Packages/Python 目录的文件将会覆盖那些存储在.sublime-package中的文件.

    在一般情况下，<executable_path>/Packages下是Sublime Text自带的，而<data_path>/Packages 的包是由用户安装的包。

    特殊的包

    有2个特殊的包: Default 和 User. Default 总是排在前面顺序的, 然后User总是后面的.Package 顺序在合并包间的文件是起作用, 举个例子 Main.sublime-menu. 任何包也许会包含一个叫做Main.sublime-menu的文件,然而不会覆盖主菜单, 反之，文件将会按照包顺序被合并.

    除了Default 和 User的包按照按字母顺序排序.

    创建一个新包

    要创建一个新包,简单的只要创建一个新的在<data_path>/Installed Packages下的目录.你可以通过 Preferences/Browse Packages菜单访问这个目录.

    覆盖压缩的包中文件

    要覆盖一个已存在包的文件,只要穿件一个同名文件到 Packages/ 目录下.

    举个例子，要覆盖 在Sublime Text自带的Python.sublime-package 包中的function.sublime-snippet 文件, 创建一个目录叫 Python 到 <data_path>/Packages 目录,然后放置你的function.sublime-snippet 文件到那儿.

    恢复到新安装时的状态

    概述

    Sublime Text 2可以通过删除你的数据目录恢复到新安装时的状态。根据你的操作系统的不同，这个目录位于:

    • OS X: ~/Library/Application Support/Sublime Text 3
    • Windows: %APPDATA%\Sublime Text 3
    • Linux: ~/.config/sublime-text-3

    要恢复到新装时的状态，你可以:

    1. 退出Sublime Text 3
    2. 删除 (或移动)数据目录,就像上文提到的。
    3. 启动Sublime Text 3

    当重启时，一个全新的数据目录会被创建，就像你装好好首次启动Sublime Text 那样。要注意的是，这样也移除了你所有的配置和安装过的包。

    OS X Lion

    In Lion, ~/Library 目录默认是隐藏的。要定位到那儿， 选中Finder中的Go/Go to Folder菜单项，然后输入 ~/Library.

    Windows

    在Windows中,缓存文件被存储在一个单独的位置, %LOCALAPPDATA%\Sublime Text 3, 通过roaming 配置文件来提高性能.

    OS X命令行

    概述

    Sublime Text 2有着一个命令行工具， subl ，来在命令行下处理文件。它可以用来打开Sublime Text 中的文件和项目，也可以作为unix工具的 编辑器 比如git和subversion。

    安装

    首要任务是建立一个symlink指向 subl 。假设你已经把Sublime Text 放入Applications目录，并且你的path环境变量里有一个 ~/bin 目录，你可以运行： 

    ln -s "/Applications/Sublime Text.app/contents/SharedSupport/bin/subl" ~/bin/subl

    移植指南

    概述

    Sublime Text 3 与Sublime Text 2在插件上有很多重要的不同之处,并且大多数插件将需要小数量的移植工作.这些变化是:

    • Python 3.3
    • 线程之外的插件
    • 异步事件
    • 受限的 begin_edit() 和 end_edit()
    • 压缩的包
    • 导入模块

    Python 3.3

    Sublime Text 3 使用Python 3.3, 而Sublime Text 2使用Python 2.6. 而且,在 OS X, 系统内建的Python不再被使用,Sublime Text使用自带的版本. Windows和 Linux也使用自带的版本,因为它们太旧了.

    线程之外的插件

    插件现在运行在一个单独的线程, plugin_host . 从一个插件作者的角度来看，他们之间没有明显的差异,除了插件host的一个崩溃将不会拖垮主程序.

    异步事件

    在Sublime Text 2中,只有set_timeout method是进程安全的. 在Sublime Text 3, 每个API method 都是进程安全的。更重要的是，有异步事件处理,来使块写非块编码更容易:

    • on_modified_async
    • on_selection_modified_async
    • on_pre_save_async
    • on_post_save_async
    • on_activated_async
    • on_deactivated_async
    • on_new_async
    • on_load_async
    • on_clone_async
    • set_timeout_async

    当写线程的代码时，牢记当你的函数执行时，缓冲去会在下面改变.

    受限的 begin_edit() 和 end_edit()

    begin_end() 和 end_edit()将不再可以访问目录, 除了在一些特殊情况下. 唯一的获取一个有效Edit对象的实例的途径是，把你的代码放到一个 TextCommand 子类中.一般，大多数代码在begin_edit() 和 end_edit()之间可以放到一个TextCommand中来 重构， 然后用run_command 调用那个TextCommand.

    这一举措消除了Edit对象晃来晃去的问题，同时保证重复的command 和 macros在正常工作。

    压缩的包

    Sublime Text 3 中的包可以从 .sublime-package (i.e., 重命名的 .zip files) 文件直接被运行, 不像Sublime Text 2,它必须运行前解压他们.

    虽然大多数的改变应当不会导致差异，但是重要的是牢记如果你在访问包中的文件这点区别.

    导入模块

    导入其他插件变得更简单和更健壮在Sublime Text 3中,并且通过一个正常的导入声明就可实现, e.g., import Default.comment 将会导入 Packages/Default/Comment.py

    启动时受限的 API 使用

    由于 plugin_host 是异步加载的, 在导入的时候不可能使用Sublime Text API.这意味着你模块中所有顶级声明一定不能调用任何 sublime 模块中的方法.在启动时, API处于休眠状态，并且沉默的忽略任何请求.
